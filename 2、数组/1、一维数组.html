1、一维数组

一维数组


C语言中并不检查数组索引是否越界，使用越界的数组下标会导致程序改变/引用其他变量的值【C信任程序员，不检查边界】

C语言不允许将数组赋值给另外一个数组不论是作为一个元素，还是一个整体。


数组定义
  • 用来存储一个相同数据类型元素的顺序集合；
  • 占据连续的内存位置；
  • 程序运行过程中数组所占的内存大小不能改变【静态内存分配模式】
    • 内存单元总数 = 数组容量 * 每个数组元素所占内存单元数
  • 数组名为常量值；
  • 数组名就是数组的首地址【第0个元素的地址】
array == &array[0];                // 两者都是常量，在程序运行过程中不会改变
array == &array;                   // &符号也可以作用于数组名【不推荐使用】



数组声明【声明后再初始化】
type 数组名[容量];                 // 此时，数组容量不可忽略
数组名[下标] = 初始值;              // 只能一个个下标赋值
  • 数组容量必须是正整数常量表达式（不能使用const声明的常量值，不能使用变量）；
  • 此时，数组元素和未初始化的普通变量一样，其中储存的都是垃圾值。




数组声明【声明并初始化】
type 数组名[容量] = { 表达式1,...,表达式n };       // 空缺的元素被赋值为默认值
type 数组名[] = { 表达式1,...,表达式n };           // 容量忽略时，将根据表达式的个数
  • 除初始化以外也不允许再使用花括号列表的形式赋值；
  • 已经初始化的数组，其数组容量可以忽略，编译器会根据实际情况自动确定数组容量，此时数组的实际大小就是初值列表中表达式的个数；
  • 表达式的个数不能大于数组容量；
  • 数组未初始化时存储的都是垃圾值；
  • 如果表达式的个数小于数组容量时，则未指定的数组元素被赋值为0；
    • 整型——默认值 0；
    • 浮点型——默认值 0.000000（取6位）；



数组声明【指定初始化】
type 数组名[容量] = {[m] = value};             // 指定初始化
  • 如果指定初始化器后面有更多的值，则这些值将被用于初始化指定元素后面的元素。
  • 如果再次指定初始化同一个元素，会更新原来的值。



数组引用
数组名[下标];                                   // 下标必定是非负整数
  • 下标可以是整形变量/表达式/常量，但必为非负整数。
  • 只能索引单个元素，无法使用切片。



获取数组的长度
int length = sizeof(arr)/sizeof(arr[0]);        // 只能在数组所在的作用域中使用。



打印数组
for(int i=0; i<len; i++){                    // len为数组的长度
    printf("%s",arr[i]);
}
  • C语言没有指定的函数可以一次性打印数组的所有值，只能使用for循环遍历。



数组的越界访问
  • 数组的定义是从低地址到高地址。栈却是从高地址向低地址递增，先定义的变量放在高地址，后定义的变量在低地址。当数组 越界访问时，会覆盖之前的先定义的变量。
